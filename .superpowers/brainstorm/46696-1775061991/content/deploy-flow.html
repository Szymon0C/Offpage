Szczegóły techniczne

  • Netlify — Deploy API (file upload, nie Git-based). Najprostszy — jeden POST z plikiem.
  • Vercel — Deployments API. Wymaga projekt setup na pierwszym deploy.
  • GitHub Pages — Tworzy/aktualizuje repo + włącza Pages. Wymaga GitHub OAuth.
  • Tokeny OAuth — przechowywane lokalnie w SQLite, zaszyfrowane kluczem z OS keychain (Tauri plugin-keychain).
  • Re-deploy — projekt pamięta provider + site ID, kolejny deploy = update tego samego site'a.
