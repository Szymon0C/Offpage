Sekcja 3: AI Pipeline — generowanie i edycja

Jak model dostaje kontekst i generuje/modyfikuje HTML

AI Prompt Pipeline
1. Generowanie od zera
System prompt
rola + reguły HTML
→
Typ strony
landing/portfolio/blog/...
→
User prompt
"kawiarnia, ciepłe kolory"
→
llama-server
stream HTML
→
Pełna strona HTML
2. Generowanie z szablonu
System prompt
→
Szablon HTML
jako kontekst
→
User prompt
"dostosuj do mojej firmy"
→
llama-server
stream HTML
→
Zmodyfikowana strona
3. Edycja przez chat
System prompt
→
Aktualny HTML
cała strona
→
Historia chatu
ostatnie N wiadomości
→
User prompt
"zmień nagłówek"
→
llama-server
→
Zaktualizowany HTML
4. Inline edit (klik na sekcję)
System prompt
→
Tylko HTML sekcji
nie cała strona!
→
User prompt
"dodaj cennik"
→
llama-server
→
Nowa sekcja → replace w DOM

Kluczowe decyzje

  • Inline edit wysyła tylko HTML sekcji — oszczędza tokeny, szybsza odpowiedź, mieści się w context window
  • Chat edit wysyła całą stronę — model widzi pełny kontekst dla globalnych zmian
  • Historia chatu jest przycinana — ostatnie N wiadomości, żeby nie przekroczyć context window (16-32k)
  • Streaming — HTML jest streamowany token po tokenie do preview, użytkownik widzi postęp na żywo
  • Undo/redo — każda zmiana AI zapisuje snapshot HTML, użytkownik może cofać
