Sekcja 2: Architektura komponentów

Jak poszczególne warstwy komunikują się ze sobą

Architektura Offpage
TAURI SHELL (Rust backend)
📁 File Manager
Zapis/odczyt projektów HTML, zarządzanie szablonami, eksport plików
🤖 AI Sidecar Manager
Start/stop llama-server, health check, request routing, streaming response
🚀 Deploy Service
OAuth tokeny, upload do Netlify/Vercel/GitHub Pages API
🔍 Hardware Detector
Detekcja GPU/RAM, wybór kwantyzacji, tier sprzętowy
💾 Project Store
SQLite — projekty, historia chatu, ustawienia, deploy config
🔄 Auto Updater
Tauri updater — aktualizacje app + szablonów
⇅ Tauri IPC (invoke / events)
FRONTEND (Webview — React + TypeScript)
💬 Chat Panel
Rozmowa z AI, history, streaming tokens, prompt templates
🖼️ Preview Manager
iframe control, postMessage bridge, responsive resize
✏️ WYSIWYG Engine
contentEditable overlay w iframe, text/color/image editing
⇅ postMessage API
PREVIEW IFRAME (izolowany sandbox)
📄 User's HTML
Wygenerowana strona — pełna izolacja CSS/JS od app UI
🎯 Offpage Helper Script
Wstrzyknięty skrypt: section detection, click handling, edit overlays, selection highlighting

Kluczowe przepływy danych

  • Generowanie: User prompt → Chat Panel → Tauri IPC → AI Sidecar (llama-server HTTP) → streaming HTML → Preview iframe
  • Inline edit: Klik na sekcję w iframe → postMessage → Chat Panel (auto-prompt z kontekstem sekcji) → AI → zaktualizowany HTML
  • WYSIWYG: Klik na tekst w iframe → contentEditable aktywny → bezpośrednia edycja → postMessage z nowym HTML do app
  • Deploy: Aktualny HTML → Tauri → Deploy Service → Netlify/Vercel/GH Pages API
